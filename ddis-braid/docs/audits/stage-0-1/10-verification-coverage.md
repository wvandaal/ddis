**Category B: Generated coherence tests -- 77 tests that DO NOT COMPILE (77 false witnesses)**

File: `/data/projects/ddis/ddis-braid/crates/braid-kernel/tests/generated_coherence_tests.rs`

This file contains 77 `proptest!` blocks generated by `braid compile --emit-tests`. Critical problems:
1. **Does not compile**: truncated at line 933 with unclosed delimiter
2. **Invalid Rust identifiers**: function names like `fn generated_:spec/inv_store_001_monotonicity` use colons and slashes, which are not valid Rust
3. **18 tautological no-op "monotonicity" tests**: check `before == after` on the same store with no mutation between them (e.g., `let before = store.datom_count(); let after = store.datom_count(); prop_assert!(before <= after);`)
4. **6 tautological no-op "immutability" tests**: snapshot store, do nothing, check still equal
5. **10 "completeness" tests calling undefined `predicate(x)`**: would fail to compile even with valid function names
6. **7 "boundedness" tests calling undefined `compute_metric()`**: same issue
7. **Tests claim coverage of 44 distinct spec elements** across 14 namespaces, but provide zero actual verification

**Category C: Kani INV-MERGE-002 identity mismatch (1 false witness)**

The spec defines INV-MERGE-002 as "Merge Cascade Completeness" (transitive resolution of merge chains). The Kani proof actually verifies frontier monotonicity (frontiers never shrink after merge).
